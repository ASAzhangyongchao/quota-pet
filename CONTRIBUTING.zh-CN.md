# 参与 QuotaPet 贡献

[English](CONTRIBUTING.md)

感谢你帮助改进 QuotaPet。修改应保持本地优先、范围小、可测试，并符合仓库内的安全与性能契约。

## 提交合并请求前

1. 使用目标明确的分支，并说明改动解决的用户问题。
2. 行为变化先更新测试，再修改实现。
3. 执行 `swift test --disable-sandbox`、`./scripts/build-app.sh` 和 `./scripts/verify-package.sh`。
4. 面向用户的英文与简体中文内容同步更新。
5. 说明对隐私、安全、性能、依赖和兼容性的影响。

## 来源与许可证要求

每次贡献都必须记录新增或改编的 **代码（code）**、**字体（font）**、**图片（image）**、**图标（icon）**、**声音（sound）**、**截图（screenshot）** 或 **依赖（dependency）** 的来源、作者与许可证，并确认其允许源码公开和二进制再分发。禁止提交：

- 未取得明确权利的产品美术、标志、字体或界面素材；
- 机密源码、账号数据、凭据、令牌、私有路径或客户内容；
- 无法说明输入来源或再分发权利的生成媒体；
- 未完成许可证、安全、隐私和维护评估的运行时依赖。

现有 Q 形桌宠与应用图标由本仓库代码生成。修改其绘制代码时必须保持原创，不得仿制 OpenAI 或其他产品受保护的美术素材。

## 品牌与产品命名

产品标题保持为 `QuotaPet`。第三方产品或模型名称只可在兼容性和额度内容中作描述性使用，不能暗示隶属或认可。修改公开名称、图标、截图或发布元数据前，请阅读 [LEGAL.zh-CN.md](LEGAL.zh-CN.md)。

## 合并请求检查清单

- [ ] 行为修复前测试能失败，修复后通过。
- [ ] 未新增密钥、个人数据、私有路径、分析上报或未批准的联网请求。
- [ ] 每项新增素材和依赖都有来源与许可证记录。
- [ ] 中英文文档含义保持一致。
- [ ] 性能仍在仓库基线范围内。
