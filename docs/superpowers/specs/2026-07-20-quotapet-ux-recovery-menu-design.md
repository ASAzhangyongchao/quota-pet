# QuotaPet UX：刷新恢复、设置可读性、弹层避让与菜单完善

## 背景

已安装 `0.1.3` 出现六类体验问题：

1. 手动刷新可长期停在「刷新中…」，按钮禁用且无错误反馈。根因包括：UI 在 `beginRefresh()` 后依赖 snapshot 收尾，但 `UsageCoordinator.refresh()` 可在 `mode == nil`、握手/读取中、或已有 operation 时提前返回且不发布 snapshot；休眠唤醒后 provider 可能已停止，界面仍保留上次 ready 快照。
2. 设置窗口固定高度且 `Form` 无滚动，Codex 信任候选一多就被裁切；整体排版乱、分区弱；窗口偏窄时路径/说明被压成「…」省略号。
3. 菜单栏左键弹出的用量卡贴近屏幕顶边，视觉上遮挡菜单栏/header。
4. 右键菜单已有退出与设置，但缺少「使用说明」「关于」；设置勾选项缺少悬停说明。
5. 界面语言目前只跟系统首选语言走，设置里无法手动切换中/英；切换后右键菜单、弹层详情、刷新态、设置文案、关于/说明应对齐同一语言。
6. 若超时后静默强制重启连接，用户会感觉「突然断一下」，体验不友好；自动恢复类动作必须先有可读提示。

## 体验原则（本轮强制）

1. **先告知，再动作**：超时恢复、强制重连等自动行为，必须先给可见提示，禁止无声直接重启。
2. **有终态**：任何「加载中」必须在超时或失败后进入可理解的成功/失败/可重试状态，禁止永久转圈。
3. **可读优先**：设置与详情文案避免无意义截断；路径可换行或可选中复制，而不是只显示「…」。
4. **分区清晰**：设置按「外观 / 连接 / 通知与启动 / 语言 / 信任 / 关于」分组，避免一坨 Form。

## 目标行为

按优先级一次落地（A → B → C → 菜单 → 语言）：

### A. 刷新卡住：先提示，再恢复

1. 手动刷新开始后，若 **20 秒内**未收到非 `.loading` 的 snapshot（含 ready / stale / unavailable / incompatible），视为卡住。
2. 卡住时的交互顺序（不可跳过提示）：
   1. 结束「刷新中」spinner，进入**超时提示态**（详情/弹层内可见文案，例如「刷新超时，即将重新连接…」；按钮可显示倒计时或「正在恢复」）。
   2. 提示展示约 **2–3 秒**（可本地化短文案），让用户看清将发生什么。
   3. 再对 provider 执行一次强制恢复：`stop` 后按当前连接模式 `recover(restartIfStopped: true)` / 等价重连。
   4. 恢复成功 → 正常 ready；失败 → 明确错误态 + 可再次点「立即刷新」。禁止静默成功/失败。
3. `refresh()` 任意提前返回路径也必须让 UI 结束 spinner（不再只转不回）。
4. 不把 20 秒超时用于后台周期刷新；仅覆盖**用户触发的手动刷新**反馈闭环。底层 RPC 请求超时（现有约 15s）保持不变。
5. 同一次手动刷新周期内，自动恢复**最多 1 次**，避免提示→重启→再超时死循环轰炸。

### B. 设置页：可滚、可排版、不乱省略

1. **滚动**：设置内容包进可滚动区域；固定合理窗口高度，超出可滚，滚动条按系统策略显示。
2. **窗口宽度**：默认内容宽度提高到约 **520–560pt**（相对现状 420），最小宽度不低于约 **480pt**，减少路径与长句被压成「…」。
3. **排版分组**（自上而下，组间分隔清晰）：
   - 外观：显示桌宠、始终置顶、鼠标穿透（各项旁 `?` 悬停说明）
   - 连接：实时 / 节能 + 既有帮助文案；快捷键与重置
   - 通知与启动：本地通知、登录启动（旁 `?`）
   - 语言：跟随系统 / 简体中文 / English
   - Codex 信任：候选列表（路径**允许换行**或 `textSelection`，避免单行省略号吞掉关键路径）
   - 关于与法律：非官方声明等
4. 问号覆盖：显示桌宠、始终置顶、鼠标穿透、本地用量通知、登录时启动。连接模式已有帮助文案可不重复问号。
5. Codex 信任列表过长靠滚动看全。本轮**不**改路径解析/信任门禁，也不合并重复「已拒绝」行（可另案）。

### C. 弹层不挡顶栏

1. 菜单栏左键用量弹层：锚定在状态项下方，并相对屏幕可见区域做垂直钳制，避免覆盖菜单栏条本身（顶边安全间距）。
2. 右键仍只弹出 `NSMenu`，不打开用量卡。
3. 桌宠展开详情不在本轮改定位策略（它不挂在菜单栏）。

### D. 右键菜单完善

1. 保留现有：刷新、显示桌宠、实时/节能、恢复交互、设置、退出。
2. 在设置与退出之间增加分组：
   - **使用说明**：按当前界面语言打开对应指南（`USER_GUIDE.zh-CN.md` / `USER_GUIDE.md`；打包资源不可用时的回退策略在实现计划中写清）。
   - **关于**：应用名、版本号、非官方声明（复用现有法律文案键）。
3. **退出**保持现有 `onQuit` / `⌘Q` 行为。
4. 菜单栏图标本轮**保持用量圆环**，不改为 AppIcon。

### E. 设置内中英文切换

1. 设置增加「语言」选项，三选一并持久化：
   - **跟随系统**（默认，行为与现状一致）
   - **简体中文**
   - **English**
2. 切换后**立即**刷新下列界面文案（无需重启 App）：
   - 右键菜单（含说明 / 关于 / 退出等）
   - 菜单栏用量弹层与桌宠详情（标题、额度名、剩余/已用、重置/倒计时、状态、刷新按钮态、超时提示）
   - 设置窗口标题与全部表单项、问号说明、分组标题、关于法律段
   - 「关于」对话框；「使用说明」按当前语言打开对应指南
3. 实现约束：
   - `Preferences` 存语言偏好；展示层以偏好解析出的 `AppLanguage` 为准。
   - 日期格式跟随所选语言 locale。
   - 不新增第三种语言；不改系统级 macOS 语言。

## 组件调整（预期落点）

- `UsageDetailsViewModel`：超时 → 提示态 → 再回调恢复；语言变更时重建 presentation。
- `StatusItemController` / `FloatingPetController`：刷新与超时提示/恢复衔接；弹层避让顶栏；右键菜单；订阅语言并重建文案。
- `UsageCoordinator` / `AppModel`：提前返回仍有 UI 终态；可调用的强制 recover（由提示态触发，非静默）。
- `SettingsView` / `SettingsWindowController`：更宽可滚窗口；分组排版；路径换行/可选中；问号；语言 Picker。
- `Preferences` + `AppLanguage` / `L10n` + `*.lproj`：语言偏好；超时提示/即将恢复/问号/菜单/分组文案。

## 验证

- 单元测试：刷新超时先进入提示态再触发恢复回调；提前返回不永久 spinner；自动恢复同周期最多一次；设置帮助键与分组相关键存在；语言偏好解析与中英文案覆盖；路径展示相关断言（不依赖单行 truncation 作为唯一展示）。
- 现有 Swift 测试套件通过。
- 手工：
  - 卡住刷新：先看到超时/即将重连提示，再恢复，可再点刷新；
  - 设置可滚、分组清楚、拉窄窗口时路径仍可读（换行或可选中，而非只剩「…」）；
  - 菜单栏弹层不盖菜单栏；右键说明/关于/退出可用；
  - 中英文切换后菜单、详情、刷新态、设置、超时提示即时切换。

## 非目标

- 不改桌宠造型、额度环形算法、通知策略、默认连接模式。
- 不修 Codex 候选「无法解析真实路径」根因（仅通过滚动与排版暴露完整列表）。
- 不把菜单栏图标换成 logo。
- 不引入自动更新框架；版本展示只读 `Bundle` / `VERSION`。
- 不支持除简体中文 / 英文以外的界面语言；不做运行时下载语言包。
- 不把设置改成多窗口/偏好面板 Preferences 系统面板（仍为应用内设置窗）。
