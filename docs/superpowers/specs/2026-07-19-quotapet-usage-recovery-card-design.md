# QuotaPet 用量恢复与详情卡片设计

## 背景与根因

已安装版能够发现 `/Applications/ChatGPT.app/Contents/Resources/codex`，但该文件当前不能通过自动信任门禁，因此解析结果是 `requiresConfirmation`。应用只在设置页提供确认按钮，桌宠详情和菜单栏弹窗却只显示“未找到已信任的 Codex 可执行文件”，导致用户无法从故障现场恢复连接。

无用量数据时，当前界面仍把失败快照的生成时间显示成“更新于”，容易被误认为额度重置时间。桌宠详情承载于透明 `NSPanel`，展开后的 `NSHostingView` 没有自己的实色背景、圆角和阴影，因此会与桌面内容混在一起。

## 目标行为

1. 检测到需要确认的 Codex 候选时，桌宠详情和菜单栏弹窗显示“确认并读取用量”按钮。点击即代表用户确认当前展示的本机 Codex 候选；应用保存完整信任指纹、重新校验文件身份并重建 provider。
2. 不自动执行签名无效或未确认的可执行文件，不降低路径、所有者、权限、哈希和重新校验门禁。
3. 没有成功用量数据时不显示伪“更新时间”。成功或过期数据明确标为“数据更新”，每个额度窗口明确标为“重置时间”。
4. 展开的桌宠详情使用不透明系统窗口底色、16 点圆角、细边框和窗口阴影；折叠桌宠继续保持透明且无阴影。
5. 修复版版本号为 `0.1.1`，重新打包、安装并验证真实只读 Codex 用量。

## 组件调整

- `AppComposition` 保存一次解析得到的 `resolutions`，并暴露首个 `requiresConfirmation` 候选。
- `AppDelegate` 把确认动作传给两个详情入口；确认成功后持久化 `TrustFingerprint` 并调用现有 provider 重建流程。
- `UsageDetailsPresentation` 决定时间文案：无真实窗口时不产生更新时间；有窗口时输出“数据更新”，窗口行输出“重置时间”。
- `UsagePopoverView` 仅在存在待确认候选时显示确认按钮。
- `FloatingPetController` 在展开时给 hosting view 配置实色图层、圆角、边框和阴影，折叠时恢复透明桌宠契约。

## 验证

- 单元测试先覆盖待确认候选暴露、确认回调、无数据时间隐藏、重置时间标签和展开卡片外观。
- 运行完整 Swift 测试、构建、包扫描和真实只读 Codex 集成测试。
- 重装后确认用量百分比与重置时间出现，并检查进程 CPU、内存和直接 Codex 子进程清理。

## 非目标

- 不缓存额度历史，不读取对话内容，不上传数据。
- 不绕过用户确认，也不根据固定路径直接信任签名无效的二进制。
- 不调整桌宠造型、通知策略、快捷键或刷新频率。
